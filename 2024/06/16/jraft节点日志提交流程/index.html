背景

这篇文章属于 jraft 阅读笔记的一部分, 这里主要是整理一下 jraft 中用户数据日志提交流程的代码。

日志生成流程

在 jraft 中, 会使用 Node 这个接口来表示一个 Raft 节点, 同时提供了 NodeImpl 的实现(ps: java 中似乎很少看到这类写法, 反倒是 C++ 中看到的比较多, 看样子 jraft 的最初的一批作者是 C++ 选手?)

接口的声明如上所示, task 中应该是会包含用户的数据;

完整的代码如上所示: NodeImpl#apply 接口实现的并不是非常复杂, apply 主要是将用户的 task 进行编码, 然后使用 applyQueue 将任务进行提交, jraft 中的 Node 支持阻塞任务和非阻塞任务两种, 两者主要差别主要在于 sequencer 中如何取出下一个 sequence, 对于非阻塞模式, 节点会在获取失败的时候抛出异常。

当一个节点竞选 Leader 成功之后会调用 ReplicatorGroup#addReplicator 方法建立到各个 Follower 节点之间的复制关系
