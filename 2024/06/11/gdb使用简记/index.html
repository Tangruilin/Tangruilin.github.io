gdb使用简记

发表于 2024-06-11
本文字数： 695 阅读时长 ≈ 3 分钟

背景

最近在工作中调查一个 core-dump, 很遗憾到现在还没有把这个 core 调查出来, 不过随着和师兄的交流, 自己对 gdb 的一些使用姿势也有了不小的进步, 借此机会整理一下。

gdb 的基础命令以及相关的使用

  1. bt 命令
    bt命令主要是用来查看函数的堆栈, 也包含 core dump 的堆栈, 此命令没有太多可以介绍的, 一般在调试 core 的时候第一时间使用这个命令, 可以看到出现 core 的时候的堆栈;

  2. f 命令

    f 命令也就是 frame, 可以用来切换堆栈, 我们可以使用 f ${num} 来切换到不同的堆栈以观察堆栈的临时变量, 同时可以使用 info frame 的命令查看当前堆栈的信息, 比如临时变量等;

  3. p 命令

    p 命令实际上是 print 命令, 这一命令主要是用来打印指定的变量, 这里打印的变量可以是栈变量, 也可以是 watch 监控的变量, 主要是用来查看变量的数据, 或者查看对应地址的内容(PS: 这个功能使用可视化界面不太容易做到);

  4. x 命令

    x 是用来以 16 进制方式打印, 这个命令是一个 上限很高 的命令, 用好这个命令需要对内存布局有比较好的了解。
    x 命令的使用是 x/FMT 的方式, 就像 help 中说的那样 o, x, d等命令用来表示打印的方式, b, h, w, g 用来表示以多少个 byte 为单位进行打印.
    如:
    x/4cb 0x0fff 就是打印 0x0fff 地址后方4个char
    x/4dg 0x0fff 表示的是打印 0x0fff 地址后的 4 个 double, 同时每个按照 8 字节解析

  5. b, n, s, c, list 命令
    b 用来打断点, n 可以在不进入下一个函数的情况下走向函数的下一步, s 可以在进入下一个函数的情况下走向下一步, c 会运行到下一个断点, list 则会列出附近的代码(前提是符号信息没有被 strip 出来, 或者有对应的 symbol 文件)
