Combining Buffered I/O and Direct I/O in Distributed File Systems阅读

发表于 2025-01-12 更新于 2025-02-05
本文字数： 447 阅读时长 ≈ 2 分钟

背景

这篇文章主要是介绍了 联合 buffer io 和 direct io 的一些工作, 分析了 buffer io 和 direct io 之间的优劣, 并且提出了一种联合 buffer io 和 direct io 的思路, 证明了结合两种 io 模式可以获得更好的吞吐。

对比


文章先对 bio 和 dio 做了一些对比, 从图中就可以看到, 以 256K 为转折点
, 随着 iosize 的增加, dio 的吞吐会逐渐提升, bio 的变化相对平稳, 在 iosize 超过 256K 以后, direct io 的性能会开始优于 buffer io。


论文中针对这个情况使用 perf 做了一下测试, 上图就是 perf 拿出来的结果。作者基于 IOR benchmark, 压了一共 2560 GB 的数据(很奇怪没有给出 iosize).
最终发现在 ext4 文件系统中, page cache 的耗时占用了大概 40%, 同时 direct write 能做到 buffer wirte 的 5 倍吞吐。

基于此作者提出了一个方案:
通过对一个 io 的各种条件的判断(比如: 文件是否上锁, client 上的剩余内存等, 决定一个 io 应该下发为 buffer io 还是 direct io), 这里被作者叫做 auto io 模型;
剩余的主要是针对性的一些测试

图中对比了 buffer io, direct io, 非对齐 direct io, auto io
基本可以看到整体上 auto io 的吞吐都是最好的, 相对来说是符合前面说的 direct io 和 buffer io 的差距大

思考

从文中的数据来看, io size 超过 1M 的时候才会比较明显的提升, 但是现在存储侧做测试基本都是 4K ~ 1M 的 io, 从实际应用层面来讲, 需要根据用户的 io pattern 具体分析是否能有提升
